The Problem

A CFO preparing for a board meeting shouldn't need analysts and days of effort to answer basic questions — but Workday's rigid reporting layer makes that the reality. The system is built around its own logic, not around how finance professionals think.

The Solution

A unified workspace where users ask questions in plain English and get immediate answers from live Workday data. Charts, income statements, and trial balances generate on demand. One click produces a formatted PowerPoint or Excel export — no manual formatting required.

Design Decisions

Finance professionals form questions, not queries — so the interface matches that. Output formats mirror artefacts teams already produce, fitting existing workflows. AI runs locally so financial data never leaves the machine, which is non-negotiable in a client context.

Project 02  ·  Project Finance

A project manager should spend their time making decisions — not searching for the data to make them.

Persona  ·  Project Finance Manager

Project finance teams track budgets, revenue, and deadlines across dozens of active projects — but the systems they work in weren't built to surface the right signals at the right time. I designed an AI-powered project intelligence dashboard that changes that.

0 Manual reports run
1.5 hrs Manual data formatting gone
Real-time Budget vs. actual tracking
100% Project portfolio coverage
View Details▾
The Problem

Project managers track budgets, deadlines, and revenue across dozens of active projects — but the systems they work in weren't built to surface the right signals at the right time. Getting a simple answer means building custom reports, cross-referencing data sources, and waiting — often until the window to act has already closed.

The Solution

An agentic project dashboard that gives finance teams real-time visibility across their full portfolio. Users ask questions in plain English and receive immediate answers from live data — then manage their projects directly, automatically balancing expenses to budgets without leaving the tool.

Design Decisions

The dashboard was designed to surface proactive signals, not just answer reactive questions — because in project finance, late information is often useless information. Maintaining a live project directory on startup means answers about closures and budgets are always current, and running AI locally keeps financial data within the client environment.

  • Live project directory — IDs, names, end dates, status, expense and revenue totals
  • Automated expense reclass functionality to prevent overspend relative to project budget
  • Proactive closure visibility — identifies projects closing soon and flags unrecorded revenue

Persona  ·  Workday Implementation Consultant

Every Workday Expenses implementation generates the same set of artifacts — SOW analysis, configuration workbooks, test scripts, questionnaires. I designed an AI-powered toolkit that generates all of them automatically, from the same data.

40+ Configuration items covered
0 Manual test scripts written
3 Implementation phases in one tool
70+ E2E test scripts generated
View Details▾
The Problem

Implementation consultants spend a significant portion of every engagement manually producing the same artifacts — SOW analysis, configuration workbooks, E2E test scripts — before meaningful delivery work can begin. It's slow, repetitive, and error-prone.

The Solution

An AI-powered toolkit that guides consultants through Discovery, Architecture, and Testing from one interface. Upload a SOW and scope is extracted automatically. Configure items and the tool generates workbooks, questionnaires, and E2E test scripts tailored to the actual values entered — not generic templates.

Design Decisions

The three-tab structure mirrors the standard implementation methodology so consultants work within a familiar framework. Artifacts reference actual configured values — production-ready, not starting points. Everything runs locally so client data never leaves the engagement environment.
